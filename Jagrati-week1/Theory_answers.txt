List is a mutable data type. We can modify , insert and delete elements very easily . For accessing elements indexing is used. It is slow as compared to tuples. We can use it when we have more insetion and deletion operations 
Tuples are immutable. Once created we cannot make changes. Elements can be accessed through indexing. It is faster therefore used when the major operation is accessing.
Set is a mutable datatype. It stores unique values and is unordered. 
Dictionary in python is used when we have to store key value pairs . Keys are unique and used to access the value elements .

def read_json_safely(path):
        try:
            with open(path, 'r') as file:
                data = json.load(file)
                return data
        except FileNotFoundError:
            return {}
        except json.JSONDecodeError:
            return None

import requests
import sys
url = "https://api.github.com/users/anthropics"
try:
    response = requests.get(url)
    if response.status_code == 200:
        data = response.json()
        print("Name:", data.get("name"))
        print("Public Repositories:", data.get("public_repos"))
        print("Followers:", data.get("followers"))

    else:
        print(f"Error: Request failed with status code {response.status_code}")
        sys.exit(1)
except requests.exceptions.RequestException as e:
    print(f"Error: Unable to make request -> {e}")
    sys.exit(1)

API keys are not hard – coded because if we push it on GitHub, then our personal api key will be available and accessible to others. They may misuse it, and therefore, we avoid directly mentioning the key. We use a .env file and store the key value there. Then, with the help of the os module we we load it into our main file. The code is as follows:
from dotenv import load_dotenv
import os
load_dotenv()
api_key = os.getenv("API_KEY")

from pydantic import BaseModel, Field
from typing import List

class WeatherReport(BaseModel):
    city: str = Field(..., min_length=1)
    temperature_c: float = Field(..., ge=-50, le=60)
    humidity: int = Field(..., ge=0, le=100)
    conditions: List[str]


# ---------------- VALID EXAMPLE ----------------
report = WeatherReport(
    city="Jaipur",
    temperature_c=31.4,
    humidity=42,
    conditions=["sunny", "windy"]
)
print(report)


# ---------------- INVALID EXAMPLE ----------------
    bad_report = WeatherReport(
        city="",                 # empty string not allowed
        temperature_c=80,       
        humidity=90
        conditions=["hot"]
    )
Print(bad_report)
It will fail as empty string is not allowed . Also temperature  is greater than 80

import httpx
import asyncio
async def fetch(url):
    async with httpx.AsyncClient(timeout=10) as client:
        response = await client.get(url)
        response.raise_for_status()
        return response.json()
async def main():
    data = await fetch("https://api.github.com/users/anthropics")
    print(data)
asyncio.run(main())
